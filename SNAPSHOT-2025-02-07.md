# SNAPSHOT — Estado del proyecto Stanlake Park Next  
**Fecha:** 7 de febrero de 2025

---

## 1. RESUMEN EJECUTIVO

Sitio web de **Stanlake Park Wine Estate** en Next.js 16 (App Router) con Tailwind 4, diseño editorial de lujo y flujo completo de páginas (home, vinos, tienda, tours, bodas, wine bar, visit, stay). La base técnica y la mayoría de las rutas están implementadas; faltan integraciones reales (CMS/commerce) y pulido de contenido. **Completitud estimada de la migración/desarrollo: ~70–75%.**

---

## 2. DECISIONES TÉCNICAS TOMADAS

- **Framework:** Next.js 16.1.6 con App Router y Turbopack. React 19.2.3.
- **Estilos:** Tailwind CSS 4 con configuración nativa en `app/globals.css` vía `@theme` (colores brand, cream, dark, gold; fuentes Playfair + Inter; keyframes Ken Burns).
- **Arquitectura:** Sin carpeta `src/` — rutas en `app/`, componentes en `components/layout` y `components/ui`, estado global en `context/TransitionContext.tsx`.
- **Rutas dinámicas:** Uso correcto de `params` como Promise: `const { slug } = await params` en `app/shop/[slug]/page.tsx`, `app/tours/[slug]/page.tsx`, `app/weddings/venues/[slug]/page.tsx`.
- **Navegación:** `TransitionLink` (en lugar de `next/link`) para transiciones con overlay; cursor personalizado (Ring & Dot) en `components/ui/CustomCursor.tsx`; cursor del sistema oculto en viewport ≥768px (`globals.css`).
- **Librerías clave:** Framer Motion 12.x, Lucide React, clsx + tailwind-merge, @vercel/speed-insights.
- **Desviación de documentación:** Estructura real es `app/` y `components/` en raíz; no existe `src/` ni `lib/`. README menciona `src/` — obsoleto.

## 3. CÓDIGO COMPLETADO

**Páginas y rutas:**
- **Home** (`app/page.tsx`): Hero con Ken Burns, ExperienceHub, TheCellar, EditorialPillars, Footer.
- **Our Wines** (`app/our-wines/page.tsx`): Catálogo por categorías (sparkling/still) con `WineCard`, datos mock.
- **Shop** (`app/shop/page.tsx`): Grid de vinos con `WineCard`; `app/shop/[slug]/page.tsx` detalle por slug (datos mock).
- **Tours** (`app/tours/page.tsx`), `app/tours/[slug]/page.tsx`: Listado y detalle dinámico.
- **Weddings** (`app/weddings/page.tsx`), VenueShowcase, `app/weddings/venues/[slug]/page.tsx`, `app/weddings/catering/page.tsx`, `app/weddings/enquire/page.tsx`.
- **Wine Bar** (`app/wine-bar/page.tsx`): PageHero, secciones editoriales, menú (Wine/Cheese/Coffee), CTA “Book a Table”.
- **Visit** (`app/visit/page.tsx`), **Stay** (`app/stay/page.tsx`): Páginas de sección con PageHero y contenido editorial.

**Componentes:**
- Layout: `Navbar` (modos winery/wedding), `Footer`, `Hero`, `PageHero`, `WeddingHero`, `WeddingNav`, `ExperienceHub`, `TheCellar`, `EditorialPillars`, `VenueShowcase`, `TransitionOverlay`.
- UI: `Button` (variants), `CustomCursor`, `Reveal` (scroll), `TransitionLink`, `TourCard`, `WineCard`.

**Integraciones:** Solo datos mock y URLs de Unsplash para imágenes. Sin CMS ni API de e‑commerce; preparado para sustitución posterior.

**Configuración:** `layout.tsx` con fuentes Inter/Playfair (display: swap), `TransitionProvider`, `CustomCursor`, SpeedInsights. `next.config.ts` por defecto.
